# Project LLM Wiki

## Purpose

This wiki stores durable, validated project knowledge that future agents and contributors should be able to recover from the repository.

Current repository files are authoritative when they disagree with this wiki.

## What Belongs Here

- Architecture notes that explain stable boundaries and tradeoffs.
- Domain concepts that make current code easier to understand.
- Decisions with enough context to avoid rediscovering the same options.
- Operations guidance that remains useful beyond one task.
- Feature summaries and durable ideas worth revisiting.

## What Does Not Belong Here

- Active task status, sprint state, execution checkpoints, or temporary todos.
- Secrets, credentials, auth tokens, private customer data, full logs, database exports, generated dumps, or unreviewed raw material.
- Copies of current source files that agents should read directly from the repository.
